Question #1: Identify four items that are typically included in the head section of an HTML document. Explain the purpose of each item.

Answer #1:
  1. < title >: The title tag is used to Title the page
  2. < meta >: The meta tag or metadata is typically used to specify character set,page description or keywords.
  3. < style >: The Style tag is used to define style informtion(css) and how a page should render in a browser.
  4. < link >: The Link Tag defines the relationship between the current document and an external source. The link tag is often used to link style sheets.

Question #3: Explain the difference between the < blockquote > tag and the < q > tag. Give an example of each that clearly illustrates the difference.

Answer #3:
  • < blockquote >: Defines a section that is quoted from another source.
  • < q >: Defines a short quotation.

    • Example of Each: While the < q > tag simply puts quotations between the text you want quotated, < blockquote > tag will include a website url to cite the source and the text quotated.

Question #4: Give three examples of ordered lists: one that uses the "start" attribute only, one that uses the "type" attribute only, and one that uses both.

Answer #4:

  1. < type > defines the type of the ordered list including type="1" being a numbered lists, type="A" is a numbered list with uppercase letters, type="i" is a numbered list with lowercase roman numbers.

  2. < start > tag defines in an ordered list what number to start at. Example: < ol start="50" > would start the list at #50.

    3. - Using both of these tags together you can start at any numeric or roman value. Ex: < ol type="I" start="50" >

Question #8: Explain the difference between the < del > tag and the < s > tag. Give an example of each that clearly illustrates the difference.

Answer #8:
  1. < del > Defines text that has been deleted from a document.
  2. < s > Defines text that is no longer correct.

    3. - < del > Indactes text that was in a document at one point but now no longer exists, while < s > defines text that is inaccurate in a document. Both sounding similar are actually very different.


Question #9: Explain thoroughly why HTML5 supports the < small > tag but not the < big > tag.

Answer #9:

The reason the < big > tag is no longer used in HTML5 and the < small > tag still is, is because the < small > tag still had relevant use with footnotes or copyright tags, while < big > already had many similar tags to it such as < h1 > < h2 > etc, so it was no longer needed.
